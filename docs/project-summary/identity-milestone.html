Plan: docs/identity-milestone-plan.md. Goal: turn biblioHack from a single-reader homelab app into a public, EU-facing multi-user app — registration, per-user shelves, per-user recommendations — without betraying the hexagonal layout. All six phases shipped on 2026-06-10.

Deviations from the plan (all deliberate, all documented)

Static frontend → client-side guards. The plan called for Astro SSR middleware to guard pages. The production frontend is a fully static Astro build served by serve — no SSR — so guards are a client-side AuthGate island redirecting to /login?next=…. Real enforcement was always the API's; the guard is UX. Astro pitfall captured along the way (hotfix bdfd1c2): children of a client:only island render as a static slot — compose pages as single React islands.

No event bus. Cache invalidation by shelf-fingerprint instead of BookRead/BookRated events — imports happen in the worker process, so only a cross-process-safe mechanism works. The fingerprint is one.

Mailgun EU instead of NAS SMTP. Verification/reset mail goes out via smtp.eu.mailgun.org:587 (alerts@mail.josearcos.me credential, shared with Grafana alerting); MAIL_FROM=biblioHack <no-reply@mail.josearcos.me>. The plan's "mailer already connected on the NAS" assumption didn't survive deliverability reality.

session_secret dropped. Opaque random session ids don't need a signing key.

Testing approach

Unit suites run the real FastAPI app with in-memory fakes overridden at the provider dependencies (no DB, no Redis); integration suites run real Postgres via testcontainers through the full register→verify→login HTTP flow, faking only the edges (hasher, mailer, captcha, session store, rate limiter). Two lessons from the Phase 5 gate worth remembering: test apps must override get_rate_limiter (a developer machine running Redis otherwise accumulates fixed-window counters across runs and 429s the suite), and a suite that exercises the app's real session dependencies must dispose the app's cached engine per test, or leaked asyncpg sockets surface as flaky PytestUnraisableExceptionWarning errors pinned on unrelated tests.
